# Ledger Roadmap

## Vision
A better Sharesight for Australian value investors: an event-sourced portfolio + tax engine that turns IBKR Flex data into correct CGT, franking, FX, and parcel-level tax optimisation for shares and options — accurate to the cent, deterministic by replay — starting on-device (W1-W4) and lifting verbatim to an AWS backend (W6) without a rewrite.

## Locked decisions
- **Money is always `Decimal`, never `Double`**; rounding only at the edges via bankers' rounding; all serialisation through `CanonicalJSON` (sorted keys, exact Decimal) -> deterministic, byte-identical replay (the "A6" property).
- **Event-sourced and append-only**: facts are the only truth and store native amounts only (never a derived/marked/AUD value); every read model is a materialised, version-stamped projection rebuildable by replay. FX and price marks are injected at projection time (R7/R8 firewall): changing the price feed or FX source can move market value but never cost base or tax.
- **`entityId` + `accountId` on every fact** from day one; tax grouped per entity (Individual 50% / SMSF 33⅓% CGT discount); separate tax entities are separate taxpayers. AUD reporting, AU residence hardcoded for the MVP.
- **Shares and options from W1**; option treatment per-entity = capital account default, revenue overridable; exercise/assignment links option <-> underlying (premium -> cost base/proceeds).
- **The domain core is pure Foundation-only Swift** (no UIKit/SwiftUI/SwiftData), Swift 6.2 strict concurrency, UI-independent — this is the server-lift seam. Single-user-local -> multi-user-hosted must be additive, not a rewrite.
- **Named protocol seams pre-placed at every device coupling**: `RawRecordSource`/`FactStore` (the key W6 pre-refactor), `ProjectionStore`, `QuoteSource`/`PriceOverlay`, `TokenStore`, and the `FiredAlert` notifier — each pointing at its AWS replacement.
- **W6 is a lift, not a rebuild** (grounded in `backend-migration-review.md`): keep `LedgerCore` verbatim, move storage/secrets/schedule/notify/transport to AWS behind the seams, put a thin read API + Cognito in front. **Path A (single-tenant unattended sync) before Path B (multi-tenant SaaS)** — A is a strict subset of B and is the literal pilot pain (unattended scheduled sync, no phone open). The two non-negotiable pre-refactors (FactStore protocol + ConnectionState consolidation) and closing the D83-T5 CGT-fallback hazard happen inside Path A regardless.
- **W6 acceptance bar**: end-to-end on real data with tax numbers byte-identical to the on-device build — the same hand-derived oracle (IND x SMSF, FY25 x FY26) that gated W1 gates the backend lift.
- **Honest about data gaps**: surface and fill missing cost basis rather than producing silently-wrong CGT; never silently apply the 50% discount to an unrecognised account (D83-T5 — safe conservative 0% fallback before any N>1 account growth).
