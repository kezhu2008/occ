> Task spec — **tech-lead-owned** (`company_policies/tasks/`), decomposed from milestone `milestones/W6.md`.
> Acceptance = the relevant `A#` from `specs/backend-lift.md`. The reviewer checks against **this**
> file; the developer writes tests against the named `A#`. Grounded verbatim in the real Ledger
> source tree at `/Users/dev/git/ledger/` — no invented shapes.
>
> **The acceptance bar of the whole W6 wave rides here** (`backend-migration-review.md` §4.2,
> system-design §15): *"tax numbers BYTE-IDENTICAL to the on-device build."* This task takes the
> Foundation-only core — already made server-liftable by W6-M1's `FactLogStore` seam — and runs it
> **verbatim** on AWS Lambda, proving determinism server-side against the real `.flex-local` bytes.
> It is blocked by W6-M1 (`tasks/W6-M1.md`): the server `FactLogStore` impl has nothing to conform to,
> and `LedgerStore` has no protocol to reach the log through, until that seam lands.

---

Goal: Package the **unmodified** `LedgerCore` for Linux/Lambda and prove the server replays the real
IBKR bytes **byte-identical to the device** — same `factDigest`, same every tax cell. A pure
packaging + proof task: **no fact changes, no projection-output changes, no tax movement.** The core
compiles verbatim into a Lambda because it imports Foundation only (`domain-core.md` A25 / R27,
D4/D30); only the `#if canImport(BackgroundTasks)` device shell drops and the server collaborators are
wired in behind the W6-M1 seams.

Spec (exactly what to build — the reviewer checks against THIS):

**1. Build `LedgerCore` as a Linux/Lambda target (the lift, not a rebuild).**
- Add a Linux/Lambda build target (container image; the AWS Swift runtime) that depends on **nothing**
  beyond Foundation — the `Package.swift` D30 note already covers the Linux build. No source change to
  `LedgerCore`: the `Fact`/`FactKind` taxonomy, `Reconciler`, `LotLedger`, every `Projection`, every
  tax engine (`CGTEngine`, `Div775Engine`, `FXGainEngine`, `FrankingEngine`, `FITOEngine`,
  `OptionTaxEngine`, `WashSaleDetector`, `EOYSummary`), `DailyPLCalculator`, and `CanonicalJSON`
  compile **as-is**.
- `PortabilityLockTests` (`domain-core.md` A25) is the gate that this stays Foundation-only — adding
  any UIKit/SwiftUI/SwiftData/CryptoKit/`BackgroundTasks` import to the lifted graph FAILs the build.
  Keep the hand-vendored pure-Foundation `SHA256.swift` (no CryptoKit, D4).

**2. Bind the server `FactLogStore` implementation behind the W6-M1 seam.**
- The server durable-log conformer (the DynamoDB-backed `FactLogStore` from system-design §3) is the
  **only** new concrete store; it conforms to the W6-M1 `FactLogStore` protocol (the exact
  `append`/`allFacts`/`digest` contract, `backend-lift.md` R1) and exposes **no** update/delete member
  (the log stays structurally append-only, `backend-lift.md` A16). For *this* task the determinism
  proof may run the conformer in its CI/local form (a real-bytes replay store); the Dynamo wiring of
  the schema is system-design's, behind the seam.

**3. Stand up the `SyncWorker` Lambda handler as a thin shell that calls `runRefresh` verbatim.**
- The handler wires the **server** collaborators — `SecretsManagerTokenStore`, the server
  `FactLogStore` impl, the shared `QuoteService`, the SNS notifier, and a server clock — and calls
  `BackgroundRefreshController.runRefresh(...)` **verbatim**. The only delta from the device path is
  that the `#if canImport(BackgroundTasks)` device shell drops; the refresh **body** is identical
  (`backend-lift.md` A8 `unattended-scheduled-sync`: `reason=.scheduled` and `.onDemand` run the same
  `runRefresh` body to the same ingest result, the chosen instant flowing through
  `SydneyMidnightBoundary` to a deterministic snapshot key).
- The handler is a **shell only** — it adds no fold logic, no projection, no tax math; all of that is
  the unmodified core reached through the seam.

**4. Commit the real `.flex-local` bytes as a golden fixture and replay server-side in CI.**
- The committed real `.flex-local` fact set + the committed `ProjectionConfig` become the
  server-replay golden. CI runs a **cold** Lambda (container) replay and asserts the result against
  the W1 oracle (the hand-derived IND × SMSF / FY25 × FY26 `expected_tax.json` + the real-bytes Div775
  cost-base walk), **never** the projection's own sum. A determinism-canary replay on a schedule
  alarms on any digest drift (system-design §15).

**Hard constraints (the bar):**
- **R23 re-lock holds (binding; the display firewall historically called "R8").** Through the
  **server** path, display/cash/daily-P/L/snapshot/overlay/quote inputs still never enter the fact
  log, the lot ledgers, or any tax/CGT/income/FXGain/EOY/optimise/Div775 input —
  `backend-lift.md` R23 re-locks the `domain-core.md` R23 `display-never-moves-tax` invariant across
  the lift (proven non-vacuously by `backend-lift.md` A4). The only intentional tax mover remains
  `Div775Engine` (D74/D84). No tax cell moves.
- **Byte-identical-replay holds (binding).** Same facts + same `ProjectionConfig` ⇒ byte-identical
  projections, server-side exactly as on device; drop all projections + replay server-side ⇒
  byte-identical output (the on-device `byte-identical-replay` property, `domain-core.md` A3,
  preserved through the seam by `backend-lift.md` A1). No
  `Date()`/`UUID()`/`TimeZone.current`/`Locale.current`/randomness introduced (D8/spec R6).
- **No scope creep.** No source change to `LedgerCore`, no new projection, no behaviour change, no
  multi-tenant schema build (that is system-design's, behind the seam — OD3). The storage-engine
  choice stays behind the `FactLogStore` seam.
- **Reuse the existing Lambda/IaC pipeline — don't greenfield.** Reuse the `aba`/`smut-lambda` deploy
  patterns + OIDC + the `.npmrc` gotchas; the container image must build **reproducibly** in the
  project's existing pipeline.
- **Rollback = revert the branch** (clean, self-contained; no data change — the device build is
  untouched).

Acceptance criteria (binary; what the tests must prove — the `A#` are from `specs/backend-lift.md`):
- **A2 — server replay == device replay (THE wave bar, the primary gate).** A cold Lambda replay of
  the golden `.flex-local` fact set + committed `ProjectionConfig` produces a `TaxProjection`
  (EOY + CGT disposals + income + forex + `forexRealisations`) serialised via `CanonicalJSON` that is
  **byte-identical** to the on-device build by exact string equality — same `factDigest`, same every
  CGT / franking / FX / FITO / Div-775 / option cell. The FY26 Div775 line equals the committed
  real-bytes oracle **−$1.76 AUD** (D84) on the server path. **Mutation:** a 1¢ or 1-byte delta FAILs;
  a server store that reorders/drops/re-formats a fact diverges the digest ⇒ RED.
- **A1 / byte-identical-replay preserved through the seam.** The full byte-identical-replay
  rebuildability suite (the `domain-core.md` A3 property, preserved server-side by `backend-lift.md`
  A1) passes through the server `FactLogStore`: drop all projections, replay server-side ⇒
  byte-identical output. **Mutation:** a path that lets the server core bypass the protocol to touch a
  concrete store FAILs the seam test.
- **A8 — unattended-scheduled-sync runs the same body.** The `SyncWorker` handler wired with server
  collaborators (canned credential, fake transport returning the real recorded bytes, fixed clock)
  invoked `reason=.scheduled` then `.onDemand` runs the **same** `runRefresh` body to the **same**
  ingest result; the chosen instant flows through `SydneyMidnightBoundary` to a deterministic snapshot
  key. **Mutation:** a scheduled/on-demand path that diverges the ingest result FAILs.
- **A16 — append-only + canonical-order (server store).** The server `FactLogStore` exposes **only**
  `append`/`allFacts`/`digest` (no update/delete); `allFacts()` returns canonical
  `(eventDate, stableSourceID)` order; a per-tenant export round-trips to the same `digest()`.
  **Mutation:** adding any mutate/delete member, or returning facts out of canonical order, FAILs.
- **Portability lock stays green.** `PortabilityLockTests` passes — the lifted `LedgerCore` graph
  imports **Foundation only**, no UIKit/SwiftUI/SwiftData/CryptoKit/`BackgroundTasks`. Server-liftable
  by construction (`domain-core.md` A25 / R27; D4/D30).
- **R8 re-lock stays green server-side.** Through the server path, the full `TaxProjection` is
  identical across `{no daily-P/L}` vs `{daily-P/L + cash + two snapshots}` while MV/daily-P/L move;
  the negative-control leak of `snapshot.mark`/`currentFX`/`netCash` into a tax input turns RED
  (`backend-lift.md` A4 discipline). No projection output changes (R23).
- **Reproducible build proven.** The container image builds reproducibly in the existing Lambda/IaC
  pipeline (`aba`/`smut-lambda` + OIDC + `.npmrc` gotchas); a determinism-canary scheduled replay
  alarms on any digest drift.

Mutation/oracle discipline (D45/D75/D58): every new test is **RED-first + mutation-checked** (author
the assertion, break the code, watch it bite). The byte-identical assertions use the **independent

walk), **never** the projection's own sum. Gates that mutation-probe the same source file run
**serially** (D58).
