## Goal

Add the live-pull `RawRecordSource` that fetches an IBKR Flex statement directly from the **Flex Web Service** (no backend, no OAuth — W2 scope) and hands the statement bytes to the **same `FlexXMLParser`** the file path already uses. The token path and the file path must be **one pipeline fed two ways**: the new source produces a `[RawRecord]` that is value-equal to `FileRawRecordSource` over the identical statement bytes, so everything downstream (`Reconciler → [Fact] → FactStore → LotLedger → projections → tax`) is untouched. This is W2's keystone — it is what makes scheduled/unattended re-sync in W6 a storage-and-transport swap, not a logic change, because idempotency is already content-addressed (`FactID = hash(stableSourceID, kind)`, no cursor).

## Spec — exactly what to build (the reviewer checks against THIS)

The verified two-step Flex Web Service protocol with **returned-Url failover** (D32). The live preflight observed `SendRequest` answered on `ndcdyn.interactivebrokers.com` but its `<Url>` element pointed at `gdcdyn.interactivebrokers.com` (IBKR data-center failover). The source therefore **must not hardcode the GetStatement host** — it reads `<Status>`, `<ReferenceCode>`, and `<Url>` out of the SendRequest envelope and issues `GetStatement` against the **returned `Url>`**, never asserting the host.

### Files to create (`Sources/LedgerCore/Ingest/`)

- **`FlexTransport.swift`** — the injectable HTTP seam that keeps `LedgerCore` portable and the source unit-testable without a network:
  ```swift
  public protocol FlexTransport: Sendable {
      func get(_ url: URL) async throws -> Data
  }
  public struct URLSessionFlexTransport: FlexTransport { /* Foundation URLSession, the default */ }
  ```
  (This is the precise move-point if a future policy forbids networking inside the core — W6 replaces it with a server transport behind the same protocol; no caller changes.)

- **`FlexWebServiceRawRecordSource.swift`** — `struct FlexWebServiceRawRecordSource: RawRecordSource`:
  - `init(token: String, queryId: String, sendBaseURL: URL = ndcdyn…/SendRequest, transport: FlexTransport = URLSessionFlexTransport(), parser: FlexXMLParser = .init(), maxPollAttempts: Int = 12, pollDelay: @Sendable () async -> Void = …)` — the poll delay is **injected** (zeroed in tests) so correctness is deterministic given the transport's responses.
  - `func fetchRawRecords() async throws -> [RawRecord]`:
    1. `GET {sendBaseURL}?t={token}&q={queryId}&v=3` → parse the SendRequest **envelope only** (a tiny dedicated XML scan for `<Status>`/`<ErrorCode>`/`<ErrorMessage>`/`<ReferenceCode>`/`<Url>` — **not** the activity parser).
    2. On `Status == "Success"`: `GET {Url}?t={token}&q={ReferenceCode}&v=3` against the **returned `Url`** (host failover honoured), polling up to `maxPollAttempts` while the statement is still generating, awaiting `pollDelay()` between attempts.
    3. On the statement response: `parser.parse(statementData)` → `[RawRecord]`. The parse path is **identical to the file source's** — only how the bytes arrive differs.
  - **Typed errors**, mirroring `FlexParseError`'s discipline (never `fatalError`, never crash):
    ```swift
    public enum FlexServiceError: Error, Sendable {
        case authFailed                                  // bad/expired token (IBKR 1003/1019-class)
        case queryNotFound                               // unknown queryId
        case generationInProgressTimedOut               // still generating after maxPollAttempts
        case badResponse(status: String, message: String) // any other non-Success envelope
        case transport(any Error)                        // wrapped FlexTransport failure
    }
    ```

### Conformance and invariants (binding)

- **Seam conformance, no caller change (R1/D85).** `FlexWebServiceRawRecordSource` is a **sibling** of `FileRawRecordSource` under the existing `protocol RawRecordSource: Sendable { func fetchRawRecords() async throws -> [RawRecord] }`. No protocol change. `LedgerStore.ingest(source:)` ingests it as a drop-in; the same source bytes land in the per-source `FactStore` directory (D49) the file path would have written, deduped by `FactID` (the `FactStore` seam half of the W2 reconciliation acceptance).
- **Foundation-only (the W6 server-lift seam).** No UIKit/SwiftUI import; `PortabilityLockTests` must stay green. The source + `FlexTransport` compile into a Lambda by construction.
- **Token discipline (D31, binding).** The token is a request parameter only. It is **never** logged, printed, persisted, placed in an error's message, embedded in a thrown value, or committed. `FlexServiceError.badResponse(status:message:)` carries the IBKR envelope message, which the implementation must confirm never echoes `t=`. Grep-clean.
- **Determinism / no hidden clock (R3/R23 firewall).** Marks and FX from `<OpenPosition>` remain projection-time overlay, **never facts** (R3 native-only facts; R23 display never touches tax) — this source only produces `[RawRecord]`; it adds no clock, no `Locale`, no randomness. Given fixed transport responses the output is byte-stable.

## Acceptance criteria (binary; the relevant `A#` from `specs/domain-core.md`)

- **A4 (the W2 keystone — token-pull == file-import / web-source==file-source, proven offline per D38).** With a **fake `FlexTransport`** fed the **real on-disk statement bytes** (`.flex-local/U0000000_last365.xml`, read via the gitignored env-pointed path — bytes never copied into the repo, D31): `FlexWebServiceRawRecordSource.fetchRawRecords()` returns a `[RawRecord]` whose count and contents are **value-equal** to `FileRawRecordSource` over the same bytes; the two `[RawRecord]` streams reconcile to a **byte/fact-identical** canonical fact log (`D_token == D_file`) **and** identical tax numbers (EOY cells + CGT disposals) under the Company config — one `stableSourceID` id space ⇒ source-independent bytes (the `web-source==file-source-byte-identical` anchor). *Non-vacuous:* the fact set is non-empty (~the real count) and the realised-CGT surface is exercised, so identity is meaningful, not `0 == 0`. *Negative control:* flip one byte of the file-path input → digests diverge (the test would catch a real divergence).
- **A4 (idempotency — same anchor, the idempotent double-import leg).** Re-ingesting the same pulled statement yields an **identical fact count** — `FactID` dedup absorbs the overlap, **no cursor / since-date**. A double `fetchRawRecords()` → reconcile → append cycle adds zero new facts and leaves the log digest byte-identical.
- **Failover (D32).** With a fake whose SendRequest `<Url>` is on a **different host** than `sendBaseURL` (the `ndcdyn → gdcdyn` case), assert the GetStatement request targets the **returned `Url`**, not the send-base host.
- **Polling.** Fake returns "generation in progress" twice, then the statement → the source retries and succeeds. Exceeding `maxPollAttempts` throws `FlexServiceError.generationInProgressTimedOut` (no crash, deterministic — `pollDelay` zeroed in test).
- **Typed errors / no crash.** SendRequest returns a non-Success envelope: auth failure → `.authFailed`; unknown query → `.queryNotFound`; any other → `.badResponse(status:message:)`; a transport throw → `.transport(_)`. No path crashes.
- **Token never leaks (D31).** A test asserts the token string appears in no thrown value, no logged/printed/stored string; grep-clean across the source. (Non-vacuous: the assertion fails if a developer interpolates `t=` into an error message.)
- **Foundation-only.** `PortabilityLockTests` green; no Apple-UI import.

> Live network is **not** in CI — it was proven once by the preflight curl (already PASS, sandbox-disabled per the access pre-flight). CI runs entirely against the fake transport over the real bytes (D38), so the gate is deterministic, sandbox-safe, and commits no PII.

## Test strategy (TDD against the `A#` — red first, mutation-proven)

Each acceptance bullet above is a test, written **before** the implementation and seen **RED** first. The fake `FlexTransport` is a canned-response double built from the real statement: SendRequest XML (`Success` + a `ReferenceCode` + a cross-host `Url`) → optional in-progress responses → the real statement bytes. The A4 harness loads the real bytes via the env-pointed gitignored path and **skips with a clear message** when `.flex-local/` is absent (a fresh clone must not hard-fail), but runs against genuine data on the dev machine. The negative-control (one-byte flip) and the token-leak grep are the non-vacuity proofs the reviewer mutation-probes.
