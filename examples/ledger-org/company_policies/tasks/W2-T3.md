# W2-T3 — Account reconciliation oracle: rebuilt-projection vs statement totals, by Decimal EXACT equality

Goal: Build a per-period **reconciliation oracle** that proves the projections rebuilt from the fact log agree with the IBKR Flex statement's own authoritative totals — **cash balance, position quantities, and realised P&L** — by `Decimal` **exact equality** (a 1¢ delta FAILs), over the real on-disk `.flex-local` bytes through a fake transport (D38). It is **independent ground truth** (D75): each statement total is re-derived first-principles, never re-summed from the projection it checks. The oracle emits a per-period `ReconciliationReport` that feeds **RPT-1 (Portfolio Valuation)** and **RPT-3 (Holding Detail / Parcel Register)** as a data-quality attestation strip. This is a verification harness over `LedgerCore`; it adds **no** core delete/mutation API, reads no display attribute into any tax input, and moves **zero** tax cells (R8).

Spec: (the reviewer checks the implementation against THIS — exact, no guessing)

1. **`ReconciliationReport` — the per-period read model.** Add a pure Foundation-only namespace `Reconciliation` to `LedgerCore` that, given `[Fact]` + `ProjectionConfig` + the statement's authoritative totals for a `(entity, account, period)`, produces a `ReconciliationReport`:
   - `period: ReconcilePeriod` — the Flex statement window (`fromDate…toDate`, half-open at the AU-FY boundary where the period is an FY; day-granular, no `Date()` read — the window is an **explicit input**, R8/determinism).
   - `lines: [ReconcileLine]`, one per checked total, where `ReconcileLine = { metric: ReconcileMetric, derived: Money|Decimal, statement: Money|Decimal, delta: Money|Decimal, status: .match | .mismatch(delta) }`.
   - `ReconcileMetric` is a **closed** enum — `cashBalance(currency)`, `positionQty(instrumentSymbol)`, `realisedPL(currency)` — switched **exhaustively, no `default:`** at every consumer (R5 discipline).
   - `verdict: .reconciled` iff **every** line is `.match`; otherwise `.failed([ReconcileMetric])` naming each mismatched metric. The report is `Sendable`, `Hashable`, and serializes via `CanonicalJSON` (sorted keys, fixed `Decimal`) so its bytes are stable run-to-run (R10).

2. **The three checked totals — derived side comes only from the rebuilt projections.**
   - **Cash** = the `CashProjection` balance per `(entity, account, currency)` — the existing signed fold over `depositWithdrawal` + `dividend`/`distribution`/`interest`/`withholding` `nativeAmount` + `trade` `netCash` + **both** legs and the commission of each `forexConversion` (R24, D64/D70/D71). **Signed and never clamped** — a negative/margin balance (the real USD margin) compares as negative.
   - **Position qty** = net open quantity per `(entity, account, instrumentSymbol)` from the rebuilt `LotLedgerState` (`Σ parcel.remaining`, long/short = the sign on quantity, R16). Options compared at contract granularity (multiplier is **not** applied to the qty check).
   - **Realised P&L** = realised proceeds − cost over the closed `DisposalLine`s in the period, per currency, **native** (the reconciliation is to the statement's native realised figure; no FX translation in this check — FX correctness is the CGT/Div775 engines' job, kept untouched here).
   - Every derived figure is produced by a **rebuild-by-replay** (`drop → replay from facts`), so the oracle also exercises A3 replay determinism on each run — if a projection is non-deterministic, the derived side is unstable and the oracle catches it.

3. **The statement side is an INDEPENDENT first-principles oracle (D75 — binding).** The `statement` value on each line is the IBKR Flex statement's **own** authoritative figure, re-derived directly from the raw statement bytes (the parsed `<Trade netCash>` / closing-cash / realised-P&L rows), **NOT** a re-sum of the projection's own fold. A self-referential oracle (e.g. `deposits + netCash` re-summed the same way the `CashProjection` sums it) is a **reviewer FAIL** — that is precisely the blind spot that let the DPC cash bug ship (D75). For the real `.flex-local` bytes under the **Acme Company** entity the cash oracle is **AUD +25,616.73 / USD +1,269.53** (signed, post the `forexConversion` fold, the D75 dropped-legs re-derivation), and the FY26 realised-forex figure that the Div775 walk independently produces is **−$1.76 AUD** — these are the pinned ground-truth anchors the oracle reconciles to.

4. **Exact equality — zero tolerance; a 1¢ miss FAILs.** Each line's `status` is `.match` iff `derived == statement` by **`Decimal` exact equality** (full-precision `Decimal`, never `Double`, never an `abs(a−b) < ε` band, R9). There is **no tolerance constant** anywhere in this code — if a reviewer finds a `tolerance`/`epsilon`/`isApproximatelyEqual`/`rounded()` smoothing on the compare path, that is a FAIL. Cross-currency compares **trap** on a currency mismatch (the `Money` invariant), so a derived AUD line can never silently "match" a USD statement figure.

5. **Per-period report feeding RPT-1 / RPT-3.** The `ReconciliationReport` is exposed to the export pipeline as a data-quality attestation:
   - **RPT-1 (Portfolio Valuation):** a trailing attestation line — `RECONCILED` (verdict `.reconciled`) or `UNRECONCILED · <metric> Δ <delta>` (verdict `.failed`) — rendered in the report's existing `.text` note channel (no new render branch; reuses the keyword-detected note row, mirroring RPT-10/RPT-11's attestation rows in `design-spec-export.md`).
   - **RPT-3 (Holding Detail / Parcel Register):** the per-instrument `positionQty` and per-currency `realisedPL` lines for that holding's `(entity, account)` render as a small `RECONCILIATION` section under the existing PARCELS / DISPOSALS bands, each row `metric · derived · statement · Δ` with the `Δ` cell sign-colored on a mismatch.
   - The attestation is **display-only** — it carries no geometry/chart and invents no content (D27); it reads the projection outputs, never writes back to the fact log or any tax input.

6. **R8 re-lock (binding).** The reconciliation oracle is a **verification metric**. It MUST NOT enter the fact log, the lot ledgers, or any tax/CGT/income/FITO/FX-gain/Div775/EOY/optimise projection **input**. Adding it moves **zero** tax cells — proven by a byte-identical-tax assertion (the full `TaxProjection` canonical JSON is byte-identical with the oracle present vs absent), **non-vacuous + mutation-proven**: a deliberate leak of a reconciliation `delta` / `statement` figure into a tax input MUST turn the test **RED** (R23, A18 discipline). Cash tax treatment is unchanged (`depositWithdrawal` stays capital, D40; TICKET-1 not re-opened).

7. **Determinism preserved (R8).** No `Date()` / `TimeZone.current` / `Locale.current` / `UUID()` / randomness inside the oracle; the period window, the statement totals, and "the facts" are **explicit inputs**. The `ReconciliationReport` is byte-reproducible for fixed inputs (A3-class replay determinism), so re-running the oracle on the same period yields byte-identical report bytes.

8. **Foundation-only (the lift seam, R27).** `Reconciliation` imports **Foundation only** (no UIKit/SwiftUI/SwiftData) so it compiles verbatim into the W6 Lambda — the `PortabilityLockTests` import-graph assertion must stay green with this addition.

Acceptance criteria: (binary; what the test must prove — each maps to an `A#` in `specs/domain-core.md`; tolerance is **zero** unless stated)

- **AC1 — Reconcile by exact equality (proves A24 / R26, R9).** For the real `.flex-local` bytes through a fake `FlexTransport` (D38), reconcile the Acme Company `(entity, account, FY26)` period: every `ReconcileLine.status == .match` and `verdict == .reconciled`, by **`Decimal` exact equality** against the **independent** statement oracle (cash **AUD +25,616.73 / USD +1,269.53** signed; per-instrument net qty; native realised P&L). A deliberately injected **1¢** delta into any one statement total flips that line to `.mismatch` and `verdict` to `.failed([thatMetric])` — RED. The statement oracle is a first-principles re-derivation over the raw bytes, **not** the projection's own re-sum (a self-referential oracle is a FAIL).

- **AC2 — Cash leg is the signed, never-clamped fold (proves A16 / R24).** The `cashBalance` derived side equals the `CashProjection` over the same facts, signed; the USD margin line compares **negative** (never clamped to 0); a deposit-only / empty period reconciles to `0` on both sides (non-vacuous). **Mutation:** dropping the `forexConversion` fold case reproduces the pre-fix **AUD +93,627.88 / USD −47,376.22** on the derived side → the line goes `.mismatch` → RED.

- **AC3 — Derived side is rebuild-by-replay (proves A3 / R8, R11).** The oracle's derived figures are taken from projections **rebuilt from empty by replay**; running the oracle twice yields **byte-identical** `ReconciliationReport` bytes (canonical JSON). A single non-determinism leak (`Date()`/`UUID()` reached on the derived path) breaks the byte equality → RED.

- **AC4 — R8 byte-identical tax, mutation-proven (proves A18 / R23).** The full `TaxProjection` (EOY cells + CGT disposals, canonical JSON) is **byte-identical** with the reconciliation oracle present vs absent. **Negative control:** a deliberate leak of a reconciliation `delta`/`statement` figure into a CGT/EOY input turns the test **RED** (a vacuous green here is a reviewer FAIL).

- **AC5 — Closed metric vocabulary, exhaustive (proves A9 / R5).** A compile-time guard + a test over `ReconcileMetric.allCases` asserts every consumer (verdict fold, RPT-1 note, RPT-3 section) handles all metric cases with **no `default:`**; adding a metric fails to compile until every switch is updated.

- **AC6 — Foundation-only portability lock holds (proves A25 / R27).** `PortabilityLockTests` asserts the `Reconciliation` import graph is Foundation-only and the package still compiles for the host (non-iOS) target; adding any UI import FAILs the build.

- **AC7 — Per-period report renders into RPT-1 / RPT-3 (proves R26 surfacing).** A reconciled period renders a `RECONCILED` attestation note on RPT-1 and a `RECONCILIATION` band on RPT-3 (per-instrument qty + per-currency realised, with `Δ`); a failed period renders `UNRECONCILED · <metric> Δ <delta>` with the `Δ` cell sign-colored. The PDF smoke bar holds (non-empty, multi-page-capable, no crash); an empty period renders the header + empty-state, never a fabricated `0`.

Persona chain: `ledger-backend-engineer` → `ledger-backend-engineer-reviewer` → `ledger-qa-engineer` (acceptance vs A24/A16/A18; cross-task review) → **adversarial review panel** (high-stakes, below).

High-stakes: **yes** — this is a correctness oracle on money-to-the-cent; a vacuous or self-referential oracle is worse than no oracle (it manufactures false confidence — the exact failure mode of the DPC cash bug, D75). Review panel lenses (each an independent adversary; gates that **mutation-probe the same source file run SEQUENTIALLY** per D58, not concurrently):
