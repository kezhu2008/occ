## Failure modes this wave (the catalog — queryable across runs)

- **FM-1 (2026-05-29): headline bound to a sample $0.** QA's first cut of the cash anchor (**A16**) asserted the corrected cash balance against a deposit-only fixture whose foreign-currency total happened to be `$0.00`, then asserted "== 0" — so it stayed GREEN even when the `.forexConversion` fold case was dropped. A test that passes when the thing it guards is deleted is vacuous (the A16 RED-first/mutation rule, D45). Root cause: the headline assertion was bound to a degenerate sample, not to the real bytes + an INDEPENDENT oracle. The same shape nearly recurred on the architect side — a conversion leg re-DERIVED (`nativeAmount − fee`) instead of SOURCED from the IBKR `<Trade>` leg (the D70/D71 lesson), which would have made the oracle a re-sum of the projection's own fold rather than independent. **Structural fix:** every headline/acceptance assertion must be bound to the real on-disk `.flex-local` bytes + a first-principles oracle re-derived OUTSIDE the projection (D75), and carry a negative control — a mutation that drops the guarded fold MUST turn it RED (proven, not asserted). The empty/deposit-only path stays as the non-vacuous `0` control, NOT as the headline.
- **FM-2 (2026-05-29): demo residue in a shipping render path.** A backend change left a demo/sample holding wired into a value that reaches a render path (the FXC analogue of the recurring "shipping app shows demo holdings" trap, D85). Caught by `ledger-backend-engineer-reviewer`'s class-closing sweep, not by the feature test (the feature test rendered real data and never noticed the latent demo branch). Root cause: on a feature/retire edit the change touched the obvious file but did not grep the whole core/app for demo/sample residue. **Structural fix:** on any retire/rename/feature edit, grep the entire core + app for demo/sample/placeholder residue (not just the file you edited), and rely on the `WaveCodeCopyLockTests` copy-truth lock (D85) that asserts no shipping surface draws demo copy. (Recurrence: 2nd — the W1→DPC demo-retire lesson resurfaced in a new path, which is why it is promoted this wave, below.)
