# W2 — Connect your real portfolio (Flex Web Service)

Goal: Replace W1's local-file ingestion with a direct, on-device pull of the founder's **real** IBKR account(s) via the Flex Web Service, so the entire W1 app (positions, CGT, franking, FX, FITO, EOY summary, optimisation, reports) now runs on real data per entity — shares and options — with data gaps surfaced and fillable, and with the foreign-currency conversion legs captured correctly (the FXC corrective leg). No backend, no OAuth: the app calls Flex directly behind the `RawRecordSource` seam W1 left in place. This is the last pre-App-Store data milestone; the Flex parser and the entire fold/engine stack are reused verbatim from W1 — W2 only adds the **transport, the stitching, the reconciliation gate, and the forex-conversion fact**.

Success criteria (observable, testable — money compared by `Decimal` **exact equality**, never tolerance):
- **S1 — Real pull works.** `FlexWebServiceRawRecordSource` runs the two-step Flex protocol against the founder's token + Query ID: `SendRequest` (`v=3`) returns Success + a reference code, then `GetStatement` returns a `<FlexStatement>` the W1 parser consumes unchanged. The `ndcdyn`→`gdcdyn` host failover and the poll-until-ready loop (IBKR "statement generation in progress") are exercised against the live service.
- **S2 — History stitched across capped ranges.** Flex activity queries cap their date window, so full history requires several stitched pulls. Concatenating the stitched statements and reconciling to facts yields **exactly** the same fact set (same `FactID` set, same count) as the union of the individual pulls — overlapping windows dedupe on the IBKR stable ID (idempotent, A4), no fact dropped at a seam.
- **S3 — Reconciliation vs the manual file (the gate).** The tax projection computed from the **token-pulled** statement is **byte-identical** (canonical JSON, `Decimal`-exact) to the same projection computed from a **manually-downloaded `.xml` file** of the same account/period. Every EOY cell and every CGT disposal matches to the cent by exact equality against an independent oracle (A24) — if a single cell differs, W2 is not done. This is the W1 hand-derived oracle, re-applied to the real pull.
- **S4 — Option lifecycle reconciles from real rows.** Real Flex expiry and exercise/assignment rows drive the W1 lot-ledger state machine (assignment links option ↔ underlying, premium → cost base/proceeds) — verified on the founder's actual option activity, not a fixture.
- **S5 — Real data gaps surfaced + fillable.** Transfers-in / pre-IBKR history with missing cost basis are detected on the real statement and surfaced in the gaps inbox; a manual cost-basis fill resolves the gap and the affected CGT updates (A12 on real data).
- **S6 — Forex conversion captured (FXC corrective leg).** Each `<Trade assetCategory="CASH" symbol="AUD.USD">` row is modelled **once** as a first-class `FactKind.forexConversion` fact carrying both signed legs (AUD base `quantity`, USD quote `proceeds`), the commission currency, and the sourced native→AUD rate — the phantom-equity fall-through is removed (a CASH row no longer becomes a fake parcel/data-gap). Off this single fact:
  - **S6a — Corrected cash.** `CashProjection` reads **AUD +25,616.73 / USD +1,269.53** (signed, never clamped), matched by **exact equality** against an independent first-principles dropped-legs oracle re-derived over the real bytes (ΣAUD `quantity` = −67,996.79, ΣAUD commission = −14.36, ΣUSD `proceeds` = +48,645.76). This is DISPLAY only and moves **zero** tax cell (R23 re-lock, byte-identical-tax A18, mutation-proven).
  - **S6b — Division 775 forex realisation (genuinely moves tax).** A `Div775Engine` computes FRE-1 realisation gain/loss on the USD cash over a per-currency AUD-functional cost-base ledger (weighted-average default, FIFO alternative, recorded on the result) → a new assessable line `EOYSummary.div775ForexRealisedAUD` folded into `eoyNetTaxableAUD` at full value (Company entity, no CGT discount), matched by exact equality to a hand-computed FRE-1 oracle over the 13 legs — **while the share per-event CGT FX translation stays byte-identical** and there is no 12-month-rule double-count.
- **S7 — Sync-now control.** A foreground `resyncNow()` re-pulls from the **stored** Keychain token (no re-typing, D48): busy/error state, idempotent on `FactID` with a concurrent-pull guard, degrade-safe (no token → reconnect prompt, a failed pull leaves prior data intact + a token-free error), and the token is never written to any log (grep-clean, D31).
- **S8 — Seam discipline preserved.** Only `RawRecordSource`'s implementation changes (file → Flex transport); the projection fold, the tax engines, and `LedgerCore`'s Foundation-only portability lock are untouched. The full suite stays green: `swift test` (core) + app `xcodebuild test`.
